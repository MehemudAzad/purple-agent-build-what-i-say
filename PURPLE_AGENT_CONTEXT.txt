1. SPATIAL REASONING TASK OVERVIEW
================================================================================

TASK: Build 3D grid structures from natural language instructions.

GRID SYSTEM:
  - 9x9 spatial cells on the x-z plane
  - Origin (0, 0) is the CENTER, marked as "highlighted square"
  - X-axis: -400 (left) to +400 (right), increments of 100
  - Z-axis: -400 (top/behind) to +400 (bottom/front), increments of 100
  - Y-axis (height): 50 (ground), 150, 250, 350, 450 (top), increments of 100
  - Valid coordinates: x,z ∈ {-400,-300,-200,-100,0,100,200,300,400}, y ∈ {50,150,250,350,450}

GRID CORNERS:
  - Bottom-left: (-400, 0, 400)  ← "in front of you, to the left"
  - Bottom-right: (400, 0, 400)  ← "in front of you, to the right"
  - Top-right: (400, 0, -400)    ← "behind you, to the right"
  - Top-left: (-400, 0, -400)    ← "behind you, to the left"

COORDINATE OUTPUT FORMAT:
  [BUILD];Color,x,y,z;Color,x,y,z;...
  Example: [BUILD];Purple,0,50,0;Green,0,150,0;Green,0,250,0
  Rules:
    - Colors must be capitalized (Purple, Green, Red, Blue, etc.)
    - No spaces in the output
    - Include ALL blocks on the grid (original + new)
    - Coordinates are exact integers

SCORING:
  - Correct structure: +10 points
  - Incorrect structure: -10 points
  - Asking a clarification question: -5 points (only 1 question allowed per round)

================================================================================
2. DIRECTION MAPPINGS (CRITICAL FOR ACCURACY)
================================================================================

HORIZONTAL DIRECTIONS:
  - "left" / "to the left" → decrease X by 100 (X -= 100)
  - "right" / "to the right" → increase X by 100 (X += 100)
  - "in front" / "front of" → increase Z by 100 (Z += 100)
  - "behind" / "back of" → decrease Z by 100 (Z -= 100)

VERTICAL DIRECTIONS:
  - "on top" / "stack" / "above" → increase Y by 100 (Y += 100)
  - "below" / "under" → decrease Y by 100 (Y -= 100)

RELATIVE POSITIONING:
  - "Extend the line" → Continue in the same direction the line is going
  - "Place one block to the right of X" → Find X's position, add (+100, 0, 0)
  - "Starting from square Y, place blocks going towards..." → Determine direction from context

3. CURRENT PIPELINE ARCHITECTURE
================================================================================

STAGE 1: CLASSIFICATION & INTENT EXTRACTION
  Input: Natural language instruction + existing grid state
  LLM: Qwen3.5-27B (via vLLM on Kaggle H100)
  Task:
    1. Identify if instruction is ambiguous (missing color or count)
    2. Extract explicitly mentioned colors
    3. Extract explicitly mentioned counts
    4. Determine confidence in ability to build
  Output: JSON object
    {
      "ambiguity": "none" | "color" | "count",
      "confidence_in_build": 0.0-1.0,
      "explicitly_mentioned_colors": ["Purple", "Green"],
      "explicitly_mentioned_counts": {"blocks": 4, "stacks": 2},
      "missing_description": "Which color for the unspeified blocks?"
    }
  Decision Gate:
    IF ambiguity_type != "none" AND confidence < 0.6:
      → Return [ASK_QUESTION] and -5 points cost
    ELSE:
      → Proceed to Stage 2

STAGE 2: SPATIAL COORDINATE GENERATION
  Input: Full instruction + resolution (if [ASK] was taken) + START_STRUCTURE
  LLM: Qwen3.5-27B (same model)
  Task:
    1. Parse the instruction grammatically
    2. Identify all spatial operations:
       - "Extend line" → compute continuation direction
       - "Place X blocks" → determine placement positions from relative directions
       - "Stack N blocks on top" → place at current_y + (100 * N)
       - "Add to each arm" → identify arms from START_STRUCTURE, place blocks
    3. Calculate exact (x, y, z) coordinates
    4. Include original START_STRUCTURE blocks + new blocks
    5. Format output as [BUILD];...
  Output: Coordinate string
    Example: [BUILD];Red,100,50,-100;Red,100,50,0;Red,100,50,100;Red,200,50,100;Red,200,50,200

FEEDBACK LOOP & SPEAKER PROFILING
  Input: Agent output vs. ground truth
  Process:
    1. Compare coordinate-by-coordinate
    2. If CORRECT (+10 points):
       - Track this speaker's successful builds
       - Learn their color/count preferences
    3. If INCORRECT (-10 points):
       - Log error pattern (direction confusion, height error, wrong arm, etc.)
       - Analyze what went wrong
       - Update speaker profile with this attempt
  Speaker Profile Tracking:
    - color_fills: list of colors used when unspecified → infer default color
    - count_fills: list of counts used when unspecified → infer default count
    - turns_seen: number of instructions from this speaker
    - correct_builds: number of times this speaker's instruction was built correctly
    - incorrect_builds: number of times this speaker's instruction failed

5. ERROR PATTERNS & ROOT CAUSES
================================================================================

ERROR TYPE 1: Direction Confusion (Front/Behind vs Left/Right)
  Symptom: Z-axis coordinates incorrect when "front"/"behind" mentioned
  Example: Round 5 - placed Purple at Z=-100 instead of Z=0
  Root Cause: Model conflates "front of grid" with "bottom left corner"
               Model reverses direction when combining multiple axes
  Frequency: 3/7 errors include direction confusion

ERROR TYPE 2: Height Misunderstanding
  Symptom: Y-coordinate wrong (150 instead of 50, or vice versa)
  Example: Round 1 - added purple blocks at Y=150 instead of Y=50
  Root Cause: Model conflates "add to structure" with "on top of structure"
               "On top of" can mean:
               - Stack directly above (Y + 100)
               - Add to existing structure (same Y)
  Frequency: 2/7 errors include height confusion

ERROR TYPE 3: Relative Position Parsing
  Symptom: When instruction says "2 blocks from existing", model places wrong count/direction
  Example: Round 3 - "extend by adding 2 blocks" got partially correct but wrong Z
  Root Cause: Multi-step spatial reasoning (find existing, determine direction, extend) fails
  Frequency: 2/7 errors

HIGH-ACCURACY PATTERNS:
  ✓ Single block at origin: 100%
  ✓ Vertical stacking (all Y changes): 100%
  ✓ Corner placements (exact coordinates): 100%
  ✓ Simple horizontal extension (same axis): 100%

LOW-ACCURACY PATTERNS:
  ✗ Multi-directional movement (combining X and Z changes): 0%
  ✗ Relative positioning from existing structure: 0%
  ✗ "Add to arms" or "each arm" parsing: 0%

8. CURRENT LIMITATIONS & CHALLENGES
================================================================================

1. SPATIAL REASONING ACCURACY: 43%
   Challenge: LLM struggles with multi-axis coordinate transformations
   Manifest: Confuses Z-axis directions, reverses left/right when adding third dimension
   Solution Ideas:
     - Fine-tune on spatial reasoning data (requires 500+ examples)
     - Add explicit coordinate diagrams to system prompt
     - Route to a specialized spatial reasoning model (if available)

2. RELATIVE POSITIONING: 0% on complex multi-step
   Challenge: "2 blocks from existing structure in direction X" requires:
     - Parse existing structure
     - Determine structure direction
     - Extend in consistent direction
   Manifest: Model adds blocks but wrong count or wrong direction
   Solution: Add explicit "find structure direction" step in Stage 2 prompt

3. AMBIGUITY RESOLUTION: Mixed results
   Challenge: Speaker preferences are learned slowly
   Manifest: Need multiple feedback rounds to infer color/count conventions
   Solution: Could pre-initialize profiles with stronger priors

4. ARM/BRANCH PARSING: Failing
   Challenge: "Add one block to each arm" requires:
     - Identify arms from START_STRUCTURE
     - Count arms
     - Place blocks at correct positions per arm
   Manifest: Model places blocks randomly or misses some arms
   Solution: Add "analyze structure branches" subroutine to Stage 2

5. Y-COORDINATE HEIGHT CONFUSION: Common error
   Challenge: "Add to structure" vs "stack on top" vs "place at same level"
   Manifest: Wrong Y values when instruction is ambiguous
   Solution: More specific prompt wording about height preservation
